  1. Get your Architect API key: This is the ARCHITECT_API_KEY from your .env file. It's used to authenticate API calls from this dashboard. If you don't have one, check your .env file or set ARCHITECT_API_KEY=your-secret-key-here and restart the architect-api service.
  2. Save the API key: Paste it in the "API Key" field above and click "Save Key". This stores it in your browser's local storage so the dashboard can make authenticated requests.
  3. Confirm bridge heartbeat: The Meshtastic bridge service should be running and show "online" status. Check that claw-offgrid-meshtastic-bridge is running in PM2.
  4. Verify Home Assistant: Set HOME_ASSISTANT_URL (e.g., http://homeassistant.local:8123) and HOME_ASSISTANT_TOKEN (long-lived access token from HA) in your .env file.
  5. Configure entities: Set OFFGRID_LIGHT_ENTITIES, OFFGRID_SENSOR_ENTITIES, and OFFGRID_PRESENCE_ENTITIES as comma-separated lists (e.g., light.living_room,light.kitchen).
  6. Test mesh communication: Send a test mesh message (click "Send Test Mesh Ping" button) and verify it appears in the "Recent Mesh Commands" and "Recent Mesh Events" tables.

    📋 How to get your ARCHITECT_API_KEY:
    1. Open your .env file in the project root directory
    2. Look for a line that says ARCHITECT_API_KEY=...
    3. If it exists, copy the value after the = sign
    4. If it doesn't exist, add this line: ARCHITECT_API_KEY=your-secret-key-here (replace with a strong random string)
    5. Restart the architect-api service: pm2 restart claw-architect-api
    6. Paste the key value in the field above and click "Save Key"
